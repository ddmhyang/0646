이번에 바꾼 것들 (근본 원인 수정):

① 모바일 뷰포트 팽창 → body { position: fixed; inset: 0; }

기존: body, html { width: 100%; height: 100%; overflow: hidden; } — 모바일 Safari에서 내부 요소 크기에 따라 뷰포트가 늘어남
수정: body { position: fixed; inset: 0; } — 뷰포트 팽창 원천 차단
② 터치 오작동 → .btn_auth { position: absolute; inset: 0; }

기존: button { display: inline-block; } 규칙이 .btn_auth에도 적용되어 width: 100%가 제대로 안 먹힘
수정: button:not(.btn_auth) 으로 분리, .btn_auth는 position: absolute; inset: 0으로 독립
③ 구조 명확화

.container { position: fixed; inset: 0; } — 이중으로 뷰포트 고정
